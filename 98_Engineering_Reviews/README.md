# Engineering Reviews

## Purpose

Engineering Reviews capture the technical review of completed engineering work.

Every completed sprint includes one or more engineering reviews generated by Cursor and reviewed by the project architect.

These reviews are **not architecture documents**.

They record implementation quality, risks, technical debt, and future improvements.

---

## Workflow

Engineering Task

↓

Cursor Implementation

↓

Engineering Review

↓

Architect Review

↓

Decision Log

↓

Merge

---

## Review Contents

Each review should include:

- Summary
- Files Created
- Files Modified
- Database Changes
- API Changes
- Tests Added
- Assumptions
- Potential Risks
- Technical Debt
- Suggested Improvements
- Architecture Concerns
- Performance Concerns
- Security Concerns
- Future Scalability
- Questions for Architect

---

## Rule

Engineering Reviews never change architecture.

Architectural decisions require an ADR.
