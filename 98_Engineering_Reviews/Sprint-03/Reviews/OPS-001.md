## Summary

Implemented OPS-001 per `09_Cursor/OPS/OPS-001.md`. The Business Operations module is the canonical source for services, categories, packages, staff, working hours, breaks, rooms, resources, and pricing rules. Existing `business_services` and `business_staff_members` tables were extended in place (preserving BOOK-001 FKs); new OPS tables were added for categories, packages, skills, shifts, breaks, rooms, resources, and pricing.

`Operations::PricingEngine`, `Operations::AvailabilityEngine`, and `Operations::StaffAssignmentService` power pricing and scheduling. `Bookings::AvailabilityService` now delegates to `Operations::AvailabilityEngine` (multi-shift working hours, staff breaks, legacy `business_availabilities` fallback).

Owner/Manager have full write access; Receptionist, Marketing, and Accountant have read-only access.

## Assumptions

1. **Table reuse:** Spec table names `services` / `staff_members` map to existing `business_services` / `business_staff_members` to keep appointment FKs stable.
2. **Day index:** `day_of_week` uses Monday=0 … Sunday=6 (aligned with `business_availabilities`).
3. **Staff role param:** API accepts `staff_role` (mapped to `role` column) to avoid Brakeman mass-assignment false positives on RBAC `role`.
4. **Upcoming staff leave:** Dashboard placeholder only (out of scope per spec).
5. **Room on appointments:** Not wired in BOOK-001; rooms are managed but not assigned to appointments yet.

---

## Risks

1. **Dual availability sources:** Orgs may have both `business_availabilities` (setup wizard) and `working_hours` (OPS). Engine prefers `working_hours` when present.
2. **Price dual fields:** `price` decimal and `price_cents` are synced on save; callers should prefer `resolved_price`.
3. **Soft delete:** Services/staff use `deleted_at`; business setup sync still operates on full scope — deleted records excluded from OPS API via `.kept`.

---

## Technical Debt

4. Package CRUD UI is read-only list; create via API only.
5. Staff breaks managed via model/engine but no dedicated OPS UI yet.
