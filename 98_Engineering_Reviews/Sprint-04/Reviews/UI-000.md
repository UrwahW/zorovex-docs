## Architecture

### Before

- Six nearly identical feature headers
- AI Studio had polished loading/error; CRM/Operations/Bookings used plain text
- `StatCard` lived in CRM but imported by Operations
- `/dashboard` was a static placeholder with three generic `Card` components
- Hardcoded radii (`rounded-[20px]`) and surface styles duplicated across features

### After

```
src/components/
‚îú‚îÄ‚îÄ page-header.tsx          # Page chrome (default + hero)
‚îú‚îÄ‚îÄ loading-state.tsx        # Async loading
‚îú‚îÄ‚îÄ error-state.tsx          # Async errors
‚îú‚îÄ‚îÄ empty-state.tsx          # Empty collections
‚îú‚îÄ‚îÄ stat-card.tsx            # Dashboard metrics
‚îú‚îÄ‚îÄ async-page-boundary.tsx  # Page-level async wrapper
‚îî‚îÄ‚îÄ ui/                      # shadcn-style primitives
```

Feature modules keep their header file names for stable imports but delegate to shared components.

`DashboardPage` defines a local `ModuleCard` pattern using shared tokens; it can be promoted to `src/components/module-card.tsx` in a later UI task if reused elsewhere.

### Design tokens

| Token | Purpose |
|-------|---------|
| `--radius-card` | Large cards / panels (20px) |
| `--radius-panel` | Inner panels (16px) |
| `--surface-card` | Translucent card background |
| `--brand-accent` | Teal accent (`#20d6c7`) |

Existing `.zorovex-setup` theme inherits the same tokens.
