## Provider Workflows

Each workflow follows the same pipeline:

```
Webhook → Verify → Normalize → POST Zorovex → Build reply → Deliver → Respond
```

| Workflow file | Webhook path | Verification |
|---------------|--------------|--------------|
| `whatsapp-gateway.workflow.json` | `/webhook/whatsapp/:organizationId` | Meta signature + challenge |
| `instagram-gateway.workflow.json` | `/webhook/instagram/:organizationId` | Meta signature + challenge |
| `facebook-gateway.workflow.json` | `/webhook/facebook/:organizationId` | Meta signature + challenge |
| `website-chat-gateway.workflow.json` | `/webhook/website-chat/:organizationId` | Widget secret (optional) |
| `twilio-voice-gateway.workflow.json` | `/webhook/twilio-voice/:organizationId` | Twilio signature → TwiML |

Workflows are generated from `scripts/build-workflows.mjs` and importable via n8n UI or `scripts/import-workflows.sh`.

## Payload Schema

Standard incoming payload (see `schemas/incoming-payload.schema.json`):

```json
{
  "organization_id": "uuid",
  "provider": "whatsapp",
  "channel": "whatsapp",
  "conversation_id": "...",
  "external_user_id": "...",
  "customer_name": "...",
  "message_type": "text",
  "message": "...",
  "attachments": [],
  "metadata": {}
}
```

Zorovex reply shape consumed by delivery builders:

```json
{
  "reply": "...",
  "attachments": [],
  "actions": [],
  "typing": true
}
```

## Environment Variables

### n8n instance (global only)

| Variable | Purpose |
|----------|---------|
| `N8N_WEBHOOK_SECRET` | Meta/Twilio webhook verification |
| `N8N_API_KEY` | Sent as `X-Channel-Gateway-Key` to Zorovex |
| `ZOROVEX_BASE_URL` | Rails API base (e.g. `http://backend:3000`) |

### Rails backend

| Variable | Purpose |
|----------|---------|
| `CHANNEL_GATEWAY_API_KEY` | Primary gateway auth key |
| `N8N_API_KEY` | Alias accepted by channel gateway auth |

No business-specific values in n8n env. Per-organization provider tokens live in **n8n credentials** named `zorovex-{provider}-{organization_id}`.

## Assumptions

1. Webhook URLs include `organizationId` as a path parameter for tenant routing.
2. Provider credentials are configured in n8n per organization, not in Rails env.
3. Self-hosted n8n mounts `integrations/n8n` with `NODE_FUNCTION_ALLOW_EXTERNAL` for shared `require()`.
4. Media URLs are normalized and forwarded; storage remains in Zorovex (AI-006).
