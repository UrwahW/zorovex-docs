## Assumptions

- Segment rules use a JSON `conditions` array with `field`, `operator`, and `value` keys
- Duplicate detection groups contacts sharing email, phone, or WhatsApp among active contacts
- Merge is irreversible via API; source contact is archived with `merge_parent_id` set
- CSV import accepts pasted content or file body; header aliases supported (e.g. "First Name" ŌåÆ `first_name`)
- Inactive contacts: status `inactive` OR no contact in 90+ days
- Upcoming birthdays: contacts with birthday in the current calendar month

---

## Risks

- Large CSV imports run synchronously in the request cycle
- Duplicate groups may overlap when contacts share multiple identifiers
- `SegmentEngine` rule operators are extensible but not validated by a dedicated schema yet

---

## Technical Debt

- Segment rule builder UI is minimal (create with default lead rule); advanced rule editing deferred
- Export returns raw CSV via `send_data` rather than async job for large datasets
- Booking history preservation relies on `bookings_count` counter; no booking records exist yet in MVP
