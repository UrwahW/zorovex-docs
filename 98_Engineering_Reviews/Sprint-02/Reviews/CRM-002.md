## Database Changes

Migration `20240628000028_create_crm_collaboration` creates:

| Table | Purpose |
|-------|---------|
| `contact_documents` | File metadata + `storage_key`, soft delete |
| `contact_tasks` | Internal tasks with assignment, priority, due date |
| `contact_communications` | Manual communication log entries |

Files stored via Active Storage (`has_one_attached :file`); `storage_key` mirrors blob key.

## Assumptions

- Document files use Active Storage disk service; `storage_key` stores blob key per spec column.
- Communication `status` defaults to `logged` for manual entries; no provider integration.
- Task assignment validates assignee is an active organization member.
- Unified timeline uses entity `updated_at` for tasks; other items use `created_at`.
- Receptionist can create documents/tasks/communications but cannot update/delete tasks or delete documents.

---

## Risks

- Blob download URLs require correct `default_url_options` in production.
- Unified timeline may grow large; no pagination yet.
