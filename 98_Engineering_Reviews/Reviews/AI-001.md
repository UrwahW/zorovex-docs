## Database Changes

### Table: `ai_agents`

| Column | Type | Notes |
|--------|------|-------|
| id | uuid | PK |
| organization_id | uuid | FK → organizations, cascade delete |
| name | string | required |
| slug | string | required, unique per org among kept rows |
| agent_type | string | required |
| status | string | default `draft`; enum: draft, active, disabled, archived |
| is_enabled | boolean | default true |
| configuration | jsonb | default structured object (see below) |
| deleted_at | datetime | soft delete |
| created_at / updated_at | datetime | |

**Indexes:** organization_id, agent_type, status, deleted_at; partial unique index on `(organization_id, slug) WHERE deleted_at IS NULL`.

**Default configuration:**

```json
{
  "personality": {},
  "behaviour": {},
  "goals": {},
  "knowledge_sources": [],
  "escalations": {}
}
```

**Migrations:** `00022` create table; `00023` backfill receptionist for existing orgs; `00024` finalization (status migration inactive→disabled, soft delete, configuration column).

## API Changes

Base path: `/api/v1/ai/agents` (authenticated, org-scoped)

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/v1/ai/agents` | List kept agents for current org |
| GET | `/api/v1/ai/agents/:id` | Show agent (404 if deleted/missing, 403 if foreign org) |
| POST | `/api/v1/ai/agents` | Create agent (slug auto-generated, default config applied) |
| PATCH | `/api/v1/ai/agents/:id` | Partial update |
| DELETE | `/api/v1/ai/agents/:id` | Soft delete (`deleted_at` set); receptionist blocked (422) |

Serializer exposes: id, name, slug, agent_type, status, is_enabled, is_system, configuration, timestamps.

## Finalization Changes (Architect Review)

1. **Status enum** — Replaced `active`/`inactive` with platform standard `draft`, `active`, `disabled`, `archived`.
2. **Soft deletes** — `deleted_at` column; DELETE sets timestamp; all queries use `kept` scope.
3. **Default configuration** — Replaced `{}` with structured JSON via `AiAgents::DefaultConfiguration`.
4. **System receptionist constant** — `AiAgents::SYSTEM_RECEPTIONIST` in `types.rb`; frontend uses `is_system` API field, no hardcoded slug checks in UI.

---

## Assumptions

- Receptionist is the only system agent in Sprint 01; `is_system` is derived from slug + agent_type match against `SYSTEM_RECEPTIONIST`.
- Configuration sections (personality, behaviour, etc.) are empty shells until AI-002+ tasks define editing UX and validation rules per section.
- Manager read-only access is sufficient for Sprint 01; no granular AI permissions yet.
- Placeholder pages (Playground, Knowledge, Behaviour, Goals, Escalations) intentionally render “coming soon” empty states.

---

## Potential Risks

- `configuration` jsonb is schema-light; malformed payloads could be saved if validation is bypassed outside the model layer.
- Soft-deleted agents retain slug uniqueness among kept rows; re-creating an agent with the same slug after soft delete is allowed by design but may confuse users.
- No audit trail table for agent changes; only Rails logger entries on create/update/delete.

---

## Architecture Concerns

None. Implementation follows ADR-009: AI Studio stores configuration only; no runtime or LLM integration. Command/query/controller layering matches existing CRM patterns.

---

## Performance Concerns

None for current scale. Agent lists are org-scoped with indexed queries; no N+1 in list endpoint.

---

## Security Concerns

- RBAC enforced server-side (policy) and client-side (nav hidden); 403 on unauthorized API access.
- Cross-org agent access returns 403, not 404, on show/update/delete.
- System receptionist cannot be deleted.
- No secrets or provider credentials in agent records.

---

## Future Scalability

- Partial unique index supports soft delete at scale.
- `AiAgents::Types::AGENT_TYPES` ready for Voice, Sales, Marketing agents (disabled in sidebar nav).
