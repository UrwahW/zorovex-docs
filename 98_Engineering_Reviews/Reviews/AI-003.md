## API Endpoints

| Method | Path | Access |
|--------|------|--------|
| GET | `/api/v1/ai/playground/options` | Owner, Manager (read) |
| POST | `/api/v1/ai/playground/run` | Owner only |

**POST body:** `{ "message": "..." }` (max 2000 chars, non-blank)

**Response:** simulated `response`, `metadata` (mock runtime, latency, tokens, confidence, schema/config version), `knowledge`, `goals`, `escalations`, escalation flags.

## Assumptions

- Mock runtime uses keyword matching against receptionist configuration (AI-002) — not semantic understanding.
- In-memory `SessionStore` tracks server-side message pairs per org/user but UI history is client-only per spec.
- Managers can load playground and view options but cannot run simulations.
- Deterministic metadata derived from message byte sum (no randomness).
