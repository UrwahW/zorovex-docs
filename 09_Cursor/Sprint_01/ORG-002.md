## Goal

When a user registers, automatically create their organization and assign them as owner.

## Acceptance Criteria

- [ ] Registration flow creates one organization per new user
- [ ] User is linked as `owner` in `organization_members`
- [ ] Organization name defaults from user input or email domain
- [ ] Registration response includes `organizationId`
- [ ] Transaction rolls back if org creation fails
- [ ] Flow documented in `03_Backend/ORGANIZATIONS.md`
