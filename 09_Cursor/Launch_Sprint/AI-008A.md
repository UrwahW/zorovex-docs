Objective
Harden the AI Runtime for production use.
This PR does not introduce new AI capabilities.
Instead, it closes the remaining gaps between Playground and Production, strengthens deterministic behavior, removes misleading simulations, and prepares the runtime for Customer Memory (AI-009).

Unified Intent Resolution

Runtime Validation Hardening
Response validation should become strict.
For pricing, availability, deposits, policies, and refunds:
Validation failure blocks the response.
The runtime must return a deterministic fallback instead of an LLM response.
Never allow hallucinated business facts.
