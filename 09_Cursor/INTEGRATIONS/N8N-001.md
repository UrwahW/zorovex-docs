# Objective

Implement the minimal n8n workflows required to connect external communication providers to Zorovex.

Business logic must NOT exist inside n8n.

n8n acts only as an Integration Gateway.

All business decisions belong to the Zorovex backend.

# Responsibilities

Each workflow must

- Verify provider webhook
- Validate payload
- Normalize payload
- Forward to Zorovex
- Receive reply
- Deliver reply

Nothing else.

# Standard Payload

All providers must produce

{
  "organization_id": "...",
  "provider": "...",
  "channel": "...",
  "conversation_id": "...",
  "external_user_id": "...",
  "customer_name": "...",
  "message_type": "...",
  "message": "...",
  "attachments": [],
  "metadata": {}
}

No provider-specific payloads beyond this point.

---

# Outgoing Payload

Zorovex returns

{
  "reply": "...",
  "attachments": [],
  "actions": [],
  "typing": true
}

Provider adapters translate this into provider-specific APIs.
